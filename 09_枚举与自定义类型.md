# 第9章：枚举与自定义类型

## 9.1 枚举类型基础

### 什么是枚举？

枚举类型（Enumerated Type）用于定义一组命名的离散值，使代码更具可读性。

```
枚举: MachineState
├── IDLE      = 0  (空闲)
├── RUNNING   = 1  (运行)
├── PAUSED    = 2  (暂停)
└── FAULT     = 3  (故障)
```

### 枚举声明语法

```st
TYPE
    (* 基本枚举 *)
    MachineState : (
        IDLE,
        RUNNING,
        PAUSED,
        FAULT
    );

    (* 带初始值的枚举 *)
    Priority : (
        LOW    := 1,
        MEDIUM := 5,
        HIGH   := 10,
        URGENT := 20
    );

    (* 带赋值的枚举 *)
    DayOfWeek : (
        SUNDAY    := 0,
        MONDAY    := 1,
        TUESDAY   := 2,
        WEDNESDAY := 3,
        THURSDAY  := 4,
        FRIDAY    := 5,
        SATURDAY  := 6
    );
END_TYPE
```

### 枚举使用示例

```st
TYPE
    MotorDirection : (
        DIR_STOP  := 0,
        DIR_FORWARD := 1,
        DIR_REVERSE := 2
    );

    ValveState : (
        VALVE_CLOSED,
        VALVE_OPENING,
        VALVE_OPEN,
        VALVE_CLOSING,
        VALVE_FAULT
    );
END_TYPE

VAR
    currentState : MachineState := IDLE;
    direction : MotorDirection;
    valveStatus : ValveState;
END_VAR

(* 枚举赋值 *)
currentState := RUNNING;
direction := DIR_FORWARD;

(* 枚举比较 *)
IF currentState = IDLE THEN
    (* 空闲状态处理 *)
END_IF

(* 枚举用于CASE语句 *)
CASE valveStatus OF
    VALVE_CLOSED:  (* 处理关闭状态 *)
    VALVE_OPENING: (* 处理正在打开 *)
    VALVE_OPEN:    (* 处理打开状态 *)
    VALVE_CLOSING: (* 处理正在关闭 *)
    VALVE_FAULT:  (* 处理故障 *)
END_CASE
```

## 9.2 子范围类型

### 什么是子范围？

子范围类型（Subrange Type）限制了基础类型值的有效范围。

```st
TYPE
    (* INT的子范围：0到100 *)
    Percentage : INT (0..100);

    (* DINT的子范围：-1000到1000 *)
    Position : DINT (-1000..1000);

    (* USINT的子范围：1到12 *)
    Month : USINT (1..12);

    (* 只能被0或1赋值的类型 *)
    BinaryFlag : INT (0..1);

    (* 像素值：0到255 *)
    PixelValue : USINT (0..255);
END_TYPE
```

### 子范围使用示例

```st
TYPE
    Index1to10 : INT (1..10);
    Score : INT (0..100);
    Temperature : INT (-50..150);
END_TYPE

VAR
    index : Index1to10;
    score : Score;
    temp : Temperature;
END_VAR

(* 赋值 *)
index := 5;     (* 合法：5在1-10范围内 *)
score := 95;    (* 合法：95在0-100范围内 *)
temp := -20;    (* 合法：-20在-50-150范围内 *)

(* 范围检查：赋值超出范围时可能产生错误或截断 *)
(* 某些编译器会自动检查并处理 *)
```

## 9.3 类型别名

### 定义类型别名

类型别名（Type Alias）为现有类型创建新的名称。

```st
TYPE
    (* 基本类型别名 *)
    Counter : INT;
    Flag : BOOL;
    TimerValue : TIME;

    (* 数组类型别名 *)
    IntArray10 : ARRAY[1..10] OF INT;
    SensorData : ARRAY[1..8] OF REAL;

    (* 字符串类型别名 *)
    DeviceName : STRING(30);
    ErrorMessage : STRING(100);

    (* 结构体类型别名 *)
    Point2D : STRUCT
        x : REAL;
        y : REAL;
    END_STRUCT;

    RGBColor : STRUCT
        red : USINT;
        green : USINT;
        blue : USINT;
    END_STRUCT;
END_TYPE

VAR
    counter1 : Counter;
    counter2 : INT;       (* Counter和INT本质相同 *)
    pixels : SensorData;
    homePosition : Point2D;
    color : RGBColor;
END_VAR

(* 使用 *)
counter1 := 100;
homePosition.x := 10.5;
homePosition.y := 20.3;

color.red := 255;
color.green := 128;
color.blue := 0;
```

## 9.4 复杂类型定义

## 9.6 实战案例

### 案例1：状态机应用

```st
TYPE
    BatchState : (
        BATCH_IDLE,
        BATCH_INIT,
        BATCH_FILLING,
        BATCH_HEATING,
        BATCH_MIXING,
        BATCH_COOLING,
        BATCH_DRAINING,
        BATCH_COMPLETE,
        BATCH_ABORTED,
        BATCH_FAULT
    );

    BatchPhase : (
        PHASE_NONE,
        PHASE_CHARGE,
        PHASE_REACT,
        PHASE_DISCHARGE,
        PHASE_CLEAN
    );
END_TYPE

FUNCTION_BLOCK BatchController
VAR_INPUT
    start : BOOL;
    abort : BOOL;
    fault : BOOL;
END_VAR
VAR_OUTPUT
    state : BatchState;
    phase : BatchPhase;
    progress : Percent;
END_VAR
VAR
    stateVar : BatchState := BATCH_IDLE;
    phaseVar : BatchPhase := PHASE_NONE;
END_VAR

BEGIN
    CASE stateVar OF
        BATCH_IDLE:
            progress := 0;
            phaseVar := PHASE_NONE;
            IF start THEN
                stateVar := BATCH_INIT;
            END_IF

        BATCH_INIT:
            progress := 5;
            stateVar := BATCH_FILLING;

        BATCH_FILLING:
            progress := 20;
            phaseVar := PHASE_CHARGE;
            IF fillComplete THEN
                stateVar := BATCH_HEATING;
            ELSIF abort THEN
                stateVar := BATCH_ABORTED;
            END_IF

        BATCH_HEATING:
            progress := 40;
            phaseVar := PHASE_REACT;
            IF temperatureReached THEN
                stateVar := BATCH_MIXING;
            ELSIF fault THEN
                stateVar := BATCH_FAULT;
            ELSIF abort THEN
                stateVar := BATCH_ABORTED;
            END_IF

        BATCH_MIXING:
            progress := 60;
            phaseVar := PHASE_REACT;
            IF mixingComplete THEN
                stateVar := BATCH_COOLING;
            ELSIF fault THEN
                stateVar := BATCH_FAULT;
            ELSIF abort THEN
                stateVar := BATCH_ABORTED;
            END_IF

        BATCH_COOLING:
            progress := 75;
            phaseVar := PHASE_REACT;
            IF cooled THEN
                stateVar := BATCH_DRAINING;
            ELSIF abort THEN
                stateVar := BATCH_ABORTED;
            END_IF

        BATCH_DRAINING:
            progress := 90;
            phaseVar := PHASE_DISCHARGE;
            IF drained THEN
                stateVar := BATCH_COMPLETE;
            ELSIF abort THEN
                stateVar := BATCH_ABORTED;
            END_IF

        BATCH_COMPLETE:
            progress := 100;
            phaseVar := PHASE_NONE;
            IF start THEN
                stateVar := BATCH_IDLE;  (* 准备下一批 *)
            END_IF

        BATCH_ABORTED:
            progress := 0;
            phaseVar := PHASE_DISCHARGE;
            IF drained THEN
                stateVar := BATCH_IDLE;
            END_IF

        BATCH_FAULT:
            phaseVar := PHASE_NONE;
            IF reset THEN
                stateVar := BATCH_IDLE;
            END_IF
    END_CASE;

    state := stateVar;
    phase := phaseVar;
END_FUNCTION_BLOCK
```

### 案例2：报警系统

```st
TYPE
    AlarmSeverity : (
        SEV_INFO,
        SEV_WARNING,
        SEV_ERROR,
        SEV_CRITICAL
    );

    AlarmStatus : (
        ALM_INACTIVE,
        ALM_ACTIVE,
        ALM_ACKNOWLEDGED,
        ALM_CLEARED
    );

    Alarm : STRUCT
        active : BOOL;
        acknowledged : BOOL;
        severity : AlarmSeverity;
        timestamp : TIME;
        code : INT;
        message : STRING(80);
        parameter : INT;
    END_STRUCT;

    AlarmHistory : STRUCT
        records : ARRAY[1..50] OF Alarm;
        count : INT;
        nextIndex : INT;
    END_STRUCT;
END_TYPE

FUNCTION_BLOCK AlarmManager
VAR_INPUT
    code : INT;
    severity : AlarmSeverity;
    message : STRING(80);
    parameter : INT;
    acknowledge : BOOL;
    reset : BOOL;
END_VAR
VAR_OUTPUT
    activeAlarmCount : INT;
    hasCritical : BOOL;
END_VAR
VAR
    alarms : ARRAY[1..20] OF Alarm;
    history : AlarmHistory;
    i : INT;
END_VAR

BEGIN
    (* 更新报警状态 *)
    FOR i := 1 TO 20 DO
        (* 检查报警代码匹配 *)
        IF alarms[i].code = code AND alarms[i].active THEN
            (* 响应确认 *)
            IF acknowledge THEN
                alarms[i].acknowledged := TRUE;
            END_IF

            (* 响应复位 *)
            IF reset THEN
                alarms[i].active := FALSE;
                alarms[i].acknowledged := FALSE;

                (* 记录到历史 *)
                IF history.count < 50 THEN
                    history.count := history.count + 1;
                    history.records[history.count] := alarms[i];
                END_IF
            END_IF
        END_IF
    END_FOR

    (* 统计 *)
    activeAlarmCount := 0;
    hasCritical := FALSE;
    FOR i := 1 TO 20 DO
        IF alarms[i].active THEN
            activeAlarmCount := activeAlarmCount + 1;
            IF alarms[i].severity = SEV_CRITICAL THEN
                hasCritical := TRUE;
            END_IF
        END_IF
    END_FOR
END_FUNCTION_BLOCK
```

```st
TYPE

    );

        duration : TIME;
        temperature : REAL (-50.0..500.0);
        speed : INT (0..5000);
        heatingMode : HeatingMode;
        mixingProfile : MixingProfile;

        id : INT (1..9999);
        version : INT (1..99);

        totalSteps : INT (1..50);
        steps : ARRAY[1..50] OF RecipeStep;
        createdBy : STRING(30);

    END_STRUCT;

    RecipeLibrary : STRUCT
        recipes : ARRAY[1..100] OF Recipe;
        count : INT;
        selectedIndex : INT;
    END_STRUCT;

FUNCTION_BLOCK RecipeManager
VAR_INPUT
    load : BOOL;
    recipeId : INT;
    save : BOOL;
END_VAR

    currentRecipe : Recipe;
    found : BOOL;
END_VAR
VAR
    library : RecipeLibrary;
    i : INT;
END_VAR

        found := FALSE;
        FOR i := 1 TO library.count DO
            IF library.recipes[i].id = recipeId THEN
                currentRecipe := library.recipes[i];
                library.selectedIndex := i;
                found := TRUE;
                EXIT;
            END_IF
        END_FOR
    END_IF
END_FUNCTION_BLOCK
```

## 9.7 类型转换

### 枚举与其他类型转换

```st
TYPE
    MyEnum : (A, B, C, D);
END_TYPE

VAR
    enumVal : MyEnum;
    intVal : INT;
END_VAR

(* 枚举转整数 *)
intVal := INT#(enumVal);    (* A=0, B=1, C=2, D=3 *)

(* 整数转枚举 *)
enumVal := TO_ENUM(MyEnum, intVal);  (* 部分编译器支持 *)
```

### 使用类型转换函数的注意事项

```st
VAR
    realVal : REAL := 3.7;
    intVal : INT;
END_VAR

(* 截断小数部分 *)
intVal := TRUNC(realVal);  (* 3 *)

(* 四舍五入 *)
intVal := ROUND(realVal);  (* 4 *)

(* 强制转换（截断） *)
intVal := INT(realVal);    (* 3 *)
```

## 9.8 本章小结

### 类型定义总结

| 类型 | 语法 | 用途 |
|------|------|------|
| 枚举 | `(val1, val2, val3)` | 命名离散值 |
| 子范围 | `BASE(min..max)` | 限制值范围 |
| 类型别名 | `TYPE_ALIAS : INT` | 创建别名 |
| 结构体 | `STRUCT ... END_STRUCT` | 组合不同类型 |

### 枚举最佳实践

1. **使用有意义的名称**：`MotorRunning` 而非 `MR`
2. **统一命名风格**：使用大写下划线如 `STATE_RUNNING`
3. **显式赋值**：复杂场景下显式指定枚举值
4. **用于CASE**：枚举是CASE语句的理想选择

### 子范围最佳实践

1. **验证输入**：使用子范围类型防止无效值
2. **文档作用**：子范围本身就是代码文档
3. **范围合理**：确保范围覆盖所有可能值
